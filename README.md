# 飛輪 Flywheel · 電商 AI 自動化 Demo

> 從行銷到成交的 GTM 全流程自動化藍圖 —— 可實際操作的產品原型。
> 核心主張：**AI 不只執行，直接給你結論。**

一位 PM 用同一座控制台同時服務多位老闆（多租戶），把五大模組串成一個越轉越快的成長飛輪：
內容引擎、攻擊端廣告、防守端客服、通路 B2B、決策大腦。

## 線上看

- **提案頁（landing）**：`index.html` — 給客戶看的整體說明與服務方案
- **控制台 Demo**：`console.html` — 可實際點按操作的營運控制台

> 部署為 GitHub Pages 後，網址首頁即為提案頁，右上「看實際控制台」進入操作 Demo。

## 本機開啟

純前端、零相依、零建置：

```bash
node serve.js 5173          # 然後開 http://localhost:5173
# 或：python3 -m http.server 5173
# 或：直接雙擊 index.html
```

## 五大模組（皆可互動）

| 模組 | 看點 |
|---|---|
| ① 內容引擎 | 生成管線逐步跑、文章/SKU 逐字串流寫出、SEO 評分 |
| ② 攻擊端廣告 | CPA 自動開關、留言雷達生素材、預算重分配模擬器 |
| ③ 防守端客服 | 自動回覆＋貼標、負評痛點清單、瑕疵率趨勢 |
| ④ 通路 B2B | 統編登入→AI 採購建議、下單→ERP→出貨流程動畫、庫存周轉 |
| ⑤ 決策大腦 | 營收×行銷成效圖、我方 vs 競品聲量、危機預警、資源配置建議 |

## 檔案

```
index.html    提案 landing 頁
console.html  營運控制台（SPA）
styles.css    控制台設計系統
landing.css   landing 樣式
data.js       模擬資料（zh-TW）
app.js        控制台路由與互動
landing.js    landing 邏輯
serve.js      本機靜態伺服器
```

> ⚠️ 所有數字、對話、輿論皆為**展示用的模擬資料**，僅示意系統能力，非真實營運數據。
